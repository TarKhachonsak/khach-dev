## รายการงานที่ทำ

1. **เพิ่ม class `.section-sub-header`** ใน `form-inspection-location-flow2-v2.component.html` ให้ section "ผู้บันทึก/เจ้าหน้าที่ของรัฐ" ตาม pattern เดียวกับไฟล์อื่นที่มีอยู่แล้ว (`form-inspection-location-read`, `person-crime-form-v3`)

2. **Bug: `<ng-template *ngIf="...">` ไม่ render content เลย** — พบใน `form-10-officer-task.component.html` block เฉพาะของ `FLOW_05_TASK_312`
   - Root cause: `*ngIf` บน `<ng-template>` โดยตรง จะถูก Angular ห่อเป็น `ng-template` อีกชั้น (`<ng-template [ngIf]><ng-template>content</ng-template></ng-template>`) และไม่มีอะไรเรียก `createEmbeddedView` ให้ ng-template ชั้นในเลย เนื้อหาจึงไม่ render ไม่ว่า condition จะเป็นจริงหรือไม่
   - แก้: เปลี่ยนเป็น `<ng-container *ngIf="...">` แทน (content ของ ng-container เป็น DOM จริง render ได้ทันที)

3. **เพิ่มเงื่อนไข `taskCode !== 'FLOW_05_TASK_312'`** ให้ block ทั่วไปที่ render `app-person-crime-form-v3` / `app-person-crime-form` — เพราะมี block เฉพาะสำหรับ `FLOW_05_TASK_312` อยู่แล้วแยกต่างหาก ถ้าไม่กันไว้จะ render ซ้ำ 2 รอบ

4. **อ่าน Figma design ผ่าน MCP** (`get_design_context`, `get_metadata`, `get_screenshot`) เพื่อ map ว่า section ไหนใน design ตรงกับ TypeForm ไหนของ `app-person-crime-form-v3`
   - พบว่า frame ชื่อ `[312] สรุปผลการพิจารณาคำชี้แจงหรือคำแก้ข้อกล่าวหา` คือ design ของ `FLOW_05_TASK_312` โดยตรง

5. **Bug: เลือก approveValue ครั้งแรกไม่ render ต้องสลับ radio ไปมาก่อนถึงจะขึ้น** — เกิดจาก `*ngIf` วางอยู่บน `<ng-template dxTemplate="item">` (มี 2 ng-template ชื่อ "item" เดียวกัน สลับกันด้วย `*ngIf`) ทำให้ DevExtreme (`dx-form`) sync `ContentChildren(DxTemplateDirective)` ไม่ทันในรอบแรกที่ ngIf เปลี่ยนจาก false→true
   - แก้: เหลือ `<ng-template dxTemplate="item">` ตัวเดียวเสมอ แล้วย้าย `*ngIf` เข้าไปเป็น `<ng-container>` ข้างในแทน — DevExtreme เห็น template คงที่ ไม่ต้อง re-discover

6. **Bug: เลือก D ใน `app-seek-link-radio` (ฝังใน TypeForm 10) แล้วไม่ render อะไรเลย** — chain การส่งค่าขาดตอน
   - `app-seek-link-radio` มี `[(value)]="approveValue"` ในตัวเอง + emit `approveValueChange` ออกมา
   - แต่ `person-crime-form-v3.component.html` bind แบบ one-way `[approveValue]="approveValue"` เฉยๆ ไม่มี `(approveValueChange)` listener เลย ค่าที่ emit จึงหายไปกลางทาง ไม่เคยไหนขึ้นไปถึง `form-10-officer-task`
   - แก้: เพิ่ม `@Output() approveValueChange` + handler ใน `person-crime-form-v3.component.ts`, ต่อ `(approveValueChange)` ใน template ของมันเอง, แล้วเปลี่ยน `form-10-officer-task.component.html` จาก `[approveValue]="approveValue"` เป็น two-way `[(approveValue)]="approveValue"` ทั้ง 2 instance (A/C/E และ D)
   - ระหว่างแก้เจอ **typo เคสตัวอักษร `<app-person-crime-form-V3>`** (V ใหญ่) ที่ไม่ตรงกับ selector จริง (`app-person-crime-form-v3` เล็กหมด) — เดิม one-way binding ไม่ error เพราะ browser lowercase tag เองตอน runtime แต่พอเปลี่ยนเป็น two-way ที่ type-checked, `ngtsc` (Angular AOT template type checker) เช็คแบบ case-sensitive เจอว่าไม่ match component เลย fallback เป็น native DOM event (`Event` type) ทำให้ type ไม่ตรงกับ `string` — compile error ที่เพิ่งโผล่มาเพราะ type checking เข้มขึ้น ไม่ใช่ bug ใหม่
   - Build ยืนยัน: `ng build --configuration redesign` ผ่าน 0 Error

7. **พบ bug คล้ายกันอีกจุดที่ยังไม่ได้แก้ (follow-up)**: ใน `ng-container *ngIf="['FLOW_05_TASK_312'].includes(_taskFormCode)"` (บรรทัด ~698) มี `dxi-item` ข้างใน 2 อัน ("ข้อหาที่กระทำผิดทางพินัย", "พฤติการณ์กระทำความผิด") ที่ยังมีเงื่อนไข `&& !['FLOW_05_TASK_312'].includes(_taskFormCode)` ติดมาด้วย ซึ่ง**ขัดแย้งกับเงื่อนไข outer โดยตรง** (outer บังคับว่าต้องเป็น FLOW_05_TASK_312 แต่ inner ปฏิเสธค่าเดียวกัน) ทำให้ block ทั้งสองไม่มีทาง render ได้เลย — คาดว่าเป็นการ copy เงื่อนไขมาจาก block พี่น้องนอก ng-container โดยไม่ได้ตัดส่วนที่ซ้ำซ้อนออก

## ปัญหาที่เจอและวิธีแก้

- **`<ng-template *ngIf>` ไม่เท่ากับ `<ng-container *ngIf>`** — ng-template คือแค่ TemplateRef เฉยๆ ไม่ใช่ DOM node ต้องมีตัว instantiate (createEmbeddedView) เนื้อหาถึงจะขึ้นจริง ถ้าจะ toggle การแสดงผลด้วย `*ngIf` ต้องใช้ `ng-container` ไม่ใช่ `ng-template`
- **DevExtreme dxTemplate + `*ngIf` บน ng-template เดียวกัน = template discovery ช้ากว่า 1 รอบ CD** — ทางที่ทนกว่า: คง ng-template ไว้ตัวเดียว แล้วสลับเนื้อหาข้างในด้วย ng-container/ngIf แทนที่จะสลับตัว ng-template เอง
- **Two-way binding chain ขาดตอนได้ง่ายเวลามี component ซ้อนกันหลายชั้น** — ทุกชั้นที่ต้องการส่งค่ากลับขึ้นไป ต้องมีทั้ง `@Output()...Change` และมีคนฟังจริง (`(xChange)="..."` ) ไม่ใช่แค่ pass `[x]` ลงไปทางเดียว มิเช่นนั้นค่าจะ "หายกลางทาง" แม้ UI จะดูเหมือนตอบสนอง (เพราะ child มี state ของตัวเอง)
- **Case mismatch ใน custom element tag name เป็น bug แฝงที่ browser runtime ซ่อนไว้ได้** — Angular ngtsc type checker เช็ค selector แบบ case-sensitive แต่ browser lowercase tag เองตอน render จริง ทำให้ตัวพิมพ์ผิดแบบนี้ไม่ error จนกว่าจะมี strict-typed binding (เช่น two-way `[(x)]`) มาเจอเข้า
