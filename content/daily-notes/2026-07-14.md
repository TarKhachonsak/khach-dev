# 2026-07-14

## เป้าหมาย/งาน

เริ่มจากคำถามอธิบายโค้ดใน `crime-scene-location-redesign` (input `deferMapUntilRequested`, `forceShowCoordinatePicker`, getter `hasCoordinates`) แล้วขยายไปสู่การ debug และแก้บั๊กแผนที่ (Leaflet) ไม่แสดงผลในหลายจุดของฟอร์ม officer-task

## ลำดับปัญหาและวิธีแก้

1. **ปุ่ม "ระบุพิกัด" ไม่แสดงใน task flow หนึ่ง** — root cause คือ template bind property ชื่อผิด case (`[taskformcode]` แทน `[taskFormCode]`) และไม่ได้ bind `[forceShowCoordinatePicker]` เลย ทำให้ getter คืนค่า false เมื่อไม่มีพิกัดอยู่ก่อน

2. **User แก้เอง (นอกเซสชัน)**: ลบ `deferMapUntilRequested`, `forceShowCoordinatePicker`, `hasCoordinates` ออกจาก component ทั้งหมด (ยืนยันว่าตั้งใจ) — ทำให้ context ของปัญหาก่อนหน้าเปลี่ยนไป ต้อง sync ใหม่ก่อนแก้ต่อ

3. **แผนที่ไม่แสดงหลังเปิด collapsible section** — root cause: Leaflet map ถูก initialize ตอน container ยัง `display:none` (section ปิดอยู่) ทำให้คำนวณขนาดผิด และไม่มีจุดไหนเรียก `invalidateSize()` หลังเปิด section
   - Fix: เพิ่ม `invalidateSize()` ใน leaflet wrapper component, ลบการเรียก `mapComponent?.showMap()` ซ้ำใน `ngOnChanges` ของ parent (ตัวการที่ทำให้แผนที่ toggle ปิด/เปิดโดยไม่ตั้งใจ), เพิ่ม toggle handler เรียก `invalidateSize()` หลังเปิด section

4. **แผนที่ยัง blank ตอน auto-show ครั้งแรก (ไม่เกี่ยวกับ collapsible)** — เดาเป็น DevExtreme layout timing ก่อน (เพิ่ม delayed `invalidateSize()`) แต่ไม่ตรงจุด

5. **Root cause แท้จริง** (ยืนยันจาก browser console): `NG0100 ExpressionChangedAfterItHasBeenCheckedError` + `Uncaught Error: Map container not found` — component toggle `isMapVisible` ของตัวเองข้างใน `ngAfterViewInit` (หลัง view ถูก check ไปแล้ว) และ template ผูก `*ngIf="isMapVisible"` กับ div ที่เป็น container ของ Leaflet เอง — เกิด race: `L.map('map')` (lookup ด้วย string id) รันก่อนที่ Angular จะ re-render div เข้า DOM จริง
   - Fix รอบแรก: ลบ internal toggle + `*ngIf` ออกจาก template ให้ div render เสมอ, ทำ `showMap()` เป็น idempotent
   - ยังพังอยู่ (error เดิม) เพราะยังใช้ string id lookup ซึ่งไม่การันตี timing
   - Fix จริง: เปลี่ยนไปใช้ `@ViewChild('mapContainer', { static: true }) mapContainerRef: ElementRef` แล้วส่ง native element เข้า `L.map()` ตรงๆ แทน string id — `static: true` การันตีว่า element พร้อมตั้งแต่ก่อน `ngOnInit`

6. Build ผ่าน 0 Error ทุกรอบ — commit แยก 2 กลุ่มตามงาน (refactor crime-scene-location, fix leaflet-map-overview)

## Follow-up ที่ยังไม่ปิด (สำคัญ)

Leaflet wrapper component เป็น **shared component** ที่ใช้ร่วมกันในฟอร์มอื่นอีกอย่างน้อย ~25 ไฟล์ ทุกไฟล์มี pattern เดียวกัน:

```ts
onToggleMap(mapComponent: any) {
    this._isMapVisible = !this._isMapVisible;
    mapComponent.showMap();   // เดิมพึ่งพา toggle behavior
}
```

การเปลี่ยน `showMap()` จาก toggle (สร้าง/ทำลายสลับกัน) เป็น idempotent เพื่อแก้ race condition ทำให้ **ทุกฟอร์มที่ใช้ pattern นี้เสียความสามารถ "กดซ้ำเพื่อซ่อนแผนที่"** ไป — แจ้ง user แล้ว รอ decision ว่าจะจำกัดขอบเขตแค่ component เดียว หรือทำ fix แบบ backward-compatible (ซ่อน/แสดงด้วย CSS + `invalidateSize()` แทนการทำลาย/สร้างใหม่ทุกครั้ง) ให้ครอบคลุมทุก consumer ยังไม่ได้ implement — เป็นงานค้าง
