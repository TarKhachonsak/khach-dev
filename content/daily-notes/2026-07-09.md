## เป้าหมายวันนี้
Debug และแก้ bug data-binding / payload ในฟอร์ม officer-task ของโปรเจกต์ ROD Officer (`form-52-officer-task`, `form-37-officer-task`)

## ปัญหาและวิธีแก้

### 1. form-52: payload ตอน save/ยืนยัน ไม่มีข้อมูล formio ล่าสุด
- **อาการ:** field ที่ user แก้ไขในฟอร์มย่อย (formio) ไม่ถูกส่งไป backend ตอนกด "บันทึก" (SD) หรือ "ยืนยัน" (AP)
- **Root cause:** โค้ด build `params` มีบรรทัด spread `_formDataio` ถูก comment ไว้ ใช้ `formioForm: this.formioSubmission` แทน ซึ่งเป็น `@Input()` ที่ bind ครั้งเดียวตอน component init แล้วไม่เคย reassign — เป็นค่านิ่ง (stale snapshot) ไม่ใช่ค่าที่ user แก้ไขจริง (ค่าจริงอยู่ใน `_formDataio` ที่อัปเดตผ่าน `onFieldChange()`)
- **Fix:** uncomment spread `...this._formDataio` เข้า `params` โดยตรง

### 2. form-37: `Sentflow()` ส่ง allegation group เป็น array ว่างเสมอ
- **อาการ:** ค่าที่ user แก้ (เช่น จำนวนวันชำระค่าปรับ) หายไปตอนส่ง flow ไปขั้นตอนถัดไป แม้ save draft จะสำเร็จและมีค่าถูกต้อง
- **Root cause:** เช็ค `Array.isArray(x)` บนตัวแปรที่ถูก declare/assign เป็น **object เสมอ** (ไม่ใช่ array) ทำให้เงื่อนไข false ตลอด ส่งค่า `[]` ว่างทุกครั้ง — ชื่อ field ปลายทางตรงกับ array field จริงที่มีอยู่ในคลาส แต่โค้ดกลับอ้างตัวแปรผิดตัว ซ้ำร้าย array ตัวจริงก็ใช้ไม่ได้เพราะถูก set ครั้งเดียวตอนโหลดข้อมูลแล้วไม่เคย sync กับค่าที่ user แก้ไขเลย
- **Fix:** ใช้ object ที่ถูก rebuild ใหม่ทุกครั้งจาก `_formData` (มีค่าล่าสุดจริง) wrap เป็น array แทน

### 3. form-37: `DAILY_FINE_DAY` / `FINE_AMOUNT` ถูกคำนวณทับค่าจาก backend ตอนโหลดหน้า
- **อาการ:** backend ส่งค่าจริงมาให้ แต่ฟอร์มแสดงค่าที่คำนวณผิดแทน (เห็นจาก screenshot เทียบ console.log กับ UI)
- **Root cause:** field ที่ `readOnly: true` ยังผูก `onValueChanged` callback ไว้ — DevExtreme ยิง event นี้ทุกครั้งที่ค่าถูก set รวมถึงตอนโหลดข้อมูลครั้งแรก (ไม่ใช่แค่ user แก้ไข) ทำให้ callback คำนวณค่าทับข้อมูลจริงจาก backend ทันทีที่โหลดเสร็จ
- **Fix:** เพิ่ม guard flag กัน callback ทำงานระหว่างช่วงโหลดข้อมูลเริ่มต้น
