## ปัญหาที่พบและวิธีแก้

### 1. Bug: FLOW_04_TASK_007 trigger `getDetailPhinai5` โดยไม่ตั้งใจ

**ไฟล์:** `src/app/common/helper-form-inspection-location.ts`

**ปัญหา:** case `FLOW_04_TASK_007` ใน switch statement มี if-else เช็ค formValue ('A'/'B') แต่ไม่มี `return` fallback เมื่อ formValue ไม่ตรงกับทั้งสองกรณี ทำให้ JavaScript fall-through ไปยัง case ถัดไป (`FLOW_01_TASK_008`, `FLOW_04_TASK_008` ฯลฯ) ที่เรียก `getDetailPhinai5`

**สาเหตุ:** JavaScript switch-case ไม่มี implicit `return` — ถ้า if-else ทุก branch มี return แต่มี path ที่ไม่ตรง if-else จะ fall-through ต่อ

**แก้:** เพิ่ม `return { ok: true }` หลัง if-else block

### 2. Bug: ปุ่ม "ยืนยันส่งขั้นตอนถัดไป" ไม่ชิดขวาเมื่อซ่อนปุ่มร่าง (form-09)

**ไฟล์:** `src/app/components/pages/form-officer-task/form-09-officer-task/form-09-officer-task.component.html`

**ปัญหา:** Container ใช้ `justify-content-between` ซึ่งต้องการ 2 children ถึงจะดัน element ไปคนละด้าน เมื่อ `*ngIf` ซ่อนปุ่มร่าง เหลือ element เดียวจึงเรียงอยู่ซ้าย

**แก้:** เปลี่ยน class บน parent div แบบ dynamic ด้วย `[ngClass]` — ถ้าเป็น `FLOW_04_TASK_009` ใช้ `justify-content-end`, ถ้าไม่ใช่ใช้ `justify-content-between`

## การตัดสินใจ

- เลือกใช้ `[ngClass]` แทน `[class.ms-auto]` บน element ลูก เพราะควบคุม layout logic ที่ parent ระดับเดียวชัดเจนกว่า
