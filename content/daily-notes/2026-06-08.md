# 2026-06-08

## Programming Journal

สรุปงาน Angular + DevExtreme ใน [[BMA ROD Officer UI]] โดยแกนหลักคือ data flow, callback context และการลด hardcode ใน component ที่ใช้ร่วมกันหลาย flow

## สิ่งที่เกิดขึ้น

- child component ที่รับ object ผ่าน `@Input()` แก้ field แล้ว parent เห็นค่า เพราะเป็น object reference เดียวกัน
- DevExtreme callback แบบ regular method ทำให้ `this` หลุด context
- auto-fill ข้าม datasource ทำไม่ได้เมื่อไม่มี stable key เชื่อมกัน จึงควร set text field หรือปรับ data model
- shared section ควรซ่อน/แสดงด้วย condition ที่ explicit ตาม workflow
- title ของ reusable section ควร resolve จาก input เช่น `taskCode` ใน child component เพื่อลด hardcode ใน parent

## Engineering Principles

- ใช้ [[Angular Input Object Reference]] อย่างตั้งใจเมื่อแชร์ mutable form model
- ใช้ [[DevExtreme Callback Context]] กับ event handler ของ third-party widget
- ถ้า datasource ไม่มี foreign key อย่าเดา mapping จาก display text
- reusable component ควรมี default behavior และ fallback ของตัวเอง
