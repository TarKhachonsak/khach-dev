## เป้าหมาย/งานวันนี้

1. หา flow_task ทั้งหมดที่เรียกใช้ component `app-crime-scene-location-redesign`
2. วิเคราะห์และแก้ปัญหาปุ่ม "ระบุพิกัด" ไม่แสดงในบาง flow_task
3. หาสาเหตุที่ฟิลด์ `RESULT_EXPLAIN_FLAG` (form-10-flow4-officer-task) กดบันทึกแล้วส่งค่าออกไปได้ แต่ไม่ถูกบันทึกลง DB

## ปัญหาที่พบและวิธีแก้

### 1. Mapping flow_task → component

Grep หา `app-crime-scene-location-redesign` เจอ 5 host component: form-02, form-61, form-63, form-104, form-204
แล้ว cross-reference กับ `form-render.component.ts` (mapping FLOW_XX_TASK_YYY → component class) เพื่อ list flow code ทั้งหมดที่ผ่าน component นี้

### 2. ปุ่ม "ระบุพิกัด" ไม่แสดงในหลาย flow_task

- Root cause: `showCoordinatePicker` getter เดิม gate ด้วย `forceShowCoordinatePicker` (มีแค่ form-104 ที่ bind ค่านี้เข้ามา) หรือมี lat/lng อยู่แล้วในข้อมูลเท่านั้น — flow อื่นที่ไม่เคยมีพิกัดมาก่อนเลยไม่มีทางเห็นปุ่ม
- ถามยืนยัน scope กับ user → ตอบว่าต้องการให้โชว์ปุ่มนี้ **ทุก flow_task ที่ใช้ component นี้**
- แก้โดยเปลี่ยน getter ให้ `return true` เสมอ (ตัดสินใจ: ไม่ต้องทำ allow-list ต่อ flow เพราะทุก flow ต้องการเหมือนกันหมด — ยิ่งน้อย logic ยิ่ง maintain ง่าย)
- **พบภายหลังว่า repo จริงมี commit จากเพื่อนร่วมทีม (2026-07-10) ที่แก้ปัญหาเดียวกันไปแล้วด้วยแนวทางคนละแบบ**: ใช้ allow-list เฉพาะ flow code ผ่าน static array `FORCE_SHOW_COORDINATE_FLOW_CODES` — ไม่ใช่ "return true" แบบที่คุยกันในบทสนทนานี้ การแก้ไขที่คุยกันไม่ได้ persist อยู่จริงใน working tree

### 3. RESULT_EXPLAIN_FLAG ส่งค่าได้แต่ไม่ลง DB

- Grep หาทุกจุดที่ set ฟิลด์นี้เจอ 3 จุด (onSave, _buildParams, Sentflow) ทั้งหมด set เป็น `this.approveValue` เหมือนกันหมด
- ตรวจ service layer แล้วไม่มีจุดไหน strip field ทิ้งฝั่ง FE — payload มีค่าจริงตอนออกจาก browser แน่นอน
- `this.approveValue` (radio A/B/C/D) ถูกใช้ set ทั้ง `FACT_FINDING_FLAG` และ `RESULT_EXPLAIN_FLAG` พร้อมกัน โดยไม่มี field ไหนผูกกับ `RESULT_EXPLAIN_FLAG` โดยตรงเลย
- สรุป: FE ไม่ใช่จุดที่ทำค่าหาย ปัญหาน่าจะอยู่ backend (DTO ไม่ map field เข้า DB หรือ column คาดหวัง value type คนละแบบ)

## Decision ที่ตัดสินใจ

- เลือก "return true เสมอ" แทน allow-list เพราะ user ยืนยัน scope ว่าต้องการทุก flow_task — แต่พบว่าโค้ดจริงใน repo ไปคนละทางแล้ว (allow-list) ต้อง reconcile ทีหลัง
- ยังไม่ลบ dead input (`forceShowCoordinatePicker`, `taskFormCode`) — รอ confirm เพิ่มเติม
