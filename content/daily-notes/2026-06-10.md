## Programming Journal

วันนี้ได้ lesson หลักเรื่อง boolean condition, state restore หลัง load data และ rendering rule ของ `dx-form`

## สิ่งที่เกิดขึ้น

- condition แบบ `!task && approveValue === B` ทำให้ case อื่นไม่ render ทั้งที่ต้องการซ่อนเฉพาะ `task + B`
- `approveValue` ไม่ถูก restore หลัง load draft เพราะเดิม set จาก form event เท่านั้น
- plain HTML และ `dxi-item` ที่ไม่มี `ng-template dxTemplate="item"` ภายใน `dx-form` ไม่ render

## Engineering Principles

- เมื่อ requirement คือซ่อนเฉพาะ combination เดียว ให้เขียนเป็น `!(A && B)` แล้วทำ truth table
- state ที่ควบคุม UI ต้อง restore จาก backend data ใน `loadData()` ไม่พึ่ง event หลัง render อย่างเดียว
- ปุ่มหรือ custom HTML ภายใน `dx-form` ต้องอยู่ใน `dxi-item` + `dxTemplate` หรือย้ายออกนอก `dx-form`
